#### 과거 작업 내역으로 돌아가는 2가지 방법
##### 1. Reset
: 시간을 과거로 돌리는 것, 과거로 돌아간 후 이후 흔적은 지워짐
```git
[git reset -(option) 명령어]
```
- git reset -soft HEAD^ : commit한 내용만 되돌려 줌
- git reset -mixed HEAD^ : add 전으로 되돌려 줌
- git reset -hard HEAD^ : working area에서 작업한 내용까지 
##### 2. Revert
: 최신을 삭제하는 것이 아니라, 과거 작업 내역 변화를 거꾸로 수행해 ***Commit 버전을 하나 더 만드는 것***

- 명령어
	- git reset head^ : 파일 자체를 삭제하는 것이 아니라, commit 하기 전 상태로 되돌림(과거 commit 이력이 사라짐)
	- git reset --hard :
	- soft >> commit이 가능한 상태로 stage에 올라온 상태

- git revert head^ : 이전 포지션을 commit 하기 전으로 되돌림
- git revert head : 현재 포지션을(현 head 위치) commit 하기 전으로 되돌림 -> 사실 상 무의미한 명령어

# 09_Constructor(생성자)
### Constructor란?
- 특별한 method 이다.
- 반드시 method 이름이 class 이름과 같다. (method 이름이 **대문자로 시작**)
- 일반적인 method 선언과 같지만, Return Type 자리가 없다.
	- `modifier class 이름([ dataType 변수이름, ... ]) {}`
- 객체가 *생성(new)되는 시점에 딱 한번 호출*한다.+
- Java의 모든 객체는 반드시 한 개 이상의 생성자를 갖는다.
- **Overloading**이 가능하다.
- 프로그래머가 생성자를 하나도 작성하지 않으면 default 생성자가 만들어진다.
	- `modifier class 이름() {}`

### 객체가 생성되는 시점에 하는 일
- 전역 변수를 0에 준하는 값으로 초기화
	- ex: `int i; -> 0`
- 전역 변수를 명시적 초기화(프로그래머가 직접 값을 입력)
	- ex: `int i = 5;`
- 생성자 구현부 실행
