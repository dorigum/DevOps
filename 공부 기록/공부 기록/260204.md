# 메소드(Method)
> 함수 = 메소드
##### 메소드의 특징
1. 객체 안에 선언되어 객체가 가지고 있는 기능이다.
2. **반드시** `class 내부` 에 선언된다.
3. 호출해서 사용한다. (호출되기 전까지는 실행되지 않는다.)
4. 재귀 호출 가능(자기 자신 안에서 자신을 호출)
5. 메소드 마지막 구현부에서 `return` 할 수 있다.
   (특정한 값을 호출한 주체에게 `return` 한다.)
6. 재사용 목적 : 코드의 중복을 피할 수 있다.

##### 메소드 작성법
- `Modifier` : 제한자(0개 이상) -> return type
	- `void 타입` : `return` 값이 X
	- `` : 반드시 메소드 구현 부분 마지막 줄에 `return` 
	- `객체 타입`
- Accese Modifier

- `Public` : 어디서나, 아무나 접근 가능
- `package(생략)`: 같은 패키지(폴더) 내에서 접근 가능
- `Protected` :  상속 관계라면 어디서나 접근 가능
- `Private` : 자기 자신만 접근 가능

###### 매개변수(파라미터) : 호출하는 메소드가 가지고 가는 데이터
- return 값은 하나만 가질 수 있다.
  
- ex: A가 떡볶이와 순대(매개변수)를 들고 B한테 갈 때(호출할 때), B는 떡볶이와 순대를 담을 수 있는 그릇(변수)이 있어야 함
  맛있게 먹은 후(기능을 다 수행한 후) A가 집으로 돌아갈 때 물건은 1개만 가져갈 수 있다.

##### 메소드 호출 방법
1. 객체를 생성해서 접근하는 방법
	1. (클래스가 데이터 타입)
		1. Heap
		2. Stack
		3. 참조 변수
		4. **메소드 호출** `변수이름.메소드이름();`

2. `static` 붙은 메소드를 호출하는 방법
	1. 객체 생성을 하지 않고

		this.bb(3); // this는 생성된 현재 객체를 뜻함(생략 가능)
		Test
	}
	
	private void bb(int i) {
	}
	
	public static int cc(String s) {
		return 5;
	}
	
	public String dd(int i, itn j) {
		return "A";
	}

class MainApp {

		int result = Test.cc("좋아요"); // cc() 메소드 호출
