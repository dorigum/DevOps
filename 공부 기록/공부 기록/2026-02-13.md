# Exception(예외)
- 프로그램 실행 도중 예외가 발생하여 프로그램이 중단되는 것을 미리 예방하는 것 -> 예외 처리
- 예외가 발생한 부분 이외의 영역은 끝까지 수행될 수 있도록 함

* Checked Exception;
	* 컴파일 에러 -> 무조건 예외 처리
* Unchecked Exception
	* 런타임 에러 (실행 도중에 발생) -> 예외 처리 유무 선택

### 예외 처리 방법
#### 직접 처리
#### 1. try, catch, finally
```java
// case 1:
try {
// 예외 발생 가능성이 있는 코드를 작성
// 1.
// 2.
// 3.
} catch(XxxException 변수) {
     // 예외가 발생했을 때 해야할 일을 작성
} catch(XxxException 변수) {
} ...

finally {
	// 예외 발생 여부와 상관없이 무조건 실행되는 코드 작성
}

// -----------------------------------------------
// case 2:
try {

} finally {

}
```

#### 2. 던지기 (throws)
- 메소드 선언부에 throws 작성
```java
class Test {
	public void aa (/*throws*/) {
		System.out.println();
		this.bb;
		System.out.println();
	}

	public void bb (/*throws*/) {
		System.out.println();
		int re = 100 / i;
		System.out.println();
	}
}

class Main {
	public static main(String[] args) {
		Test t = new Test();
		t.aa();
		
		System.out.println("End");
	}
}
```
- ==throws 사용하는 이유==
	- 예외 처리 통합 관리(한쪽에서 모두 처리)
	- 메소드를 사용 후 발생한 예외가 무엇인지 파악하고, 이후 예외를 어떻게 처리할 것인지 결정할 수 있도록 함(ex: 처음부터 리셋, 미리 값을 설정해줌...)
- throws를 해야할 경우에 `try {...} finally {...}` 구조를 자주 사용함


#### 예외 강제로 발생 시키기
```java
throw new Exception();
```

#### 사용자 정의 Exception
```java
// 사용자 정의 예외 클래스 작성
// Exception을 상속 받아야 사용 가능
class AuthenticationException extends RuntimeException {
	public AuthenticationException() { // String message 실행을 위한 생성자 클래스
		super("인증 정보를 다시 확인하세요.");
	}
	
	public AuthenticationException(String message) {
		super(message);
	}
}

throw new AuthenticationException(); // 생성자 호출 -> RuntimeException의 기본 생성자
throw new AuthenticationException("인증 확인해주세요.");
throw new AuthenticationException("정보가 일치하지 않아요.");
```
